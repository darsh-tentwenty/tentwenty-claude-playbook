01 — Setup

Setting up Claude to work your way

Before anything, spend 5 minutes setting your preferences. This makes every conversation smarter — Claude will already know your role, tools, and communication style.

User Preferences
Settings → Profile → Preferences
Tell Claude who you are and how you work. Example: "I'm a UI/UX designer at tentwenty. I work in Figma, I hand off to developers, and I want Claude to always give me practical, actionable advice. Keep tone casual but professional. I prefer explanations that start with the big picture."
Writing Style
Settings → Profile → Style
Customize how Claude writes back to you. Choose tone (casual, formal, direct), response length (concise vs detailed), and format (prose vs lists). Designers: set "concise + visual thinking". Devs: set "technical + code-first".
Feature Toggles
Settings → Features
Enable: Web Search (for live references), Code Execution (for devs), Artifacts (to generate sharable files), and Memory (so Claude remembers your context across sessions). Keep all of these on.
Cowork (Coming)
Settings → Subscription
Once the team upgrades to the Team plan, Cowork enables Claude to automate desktop file tasks — great for batch design exports, file renaming, and organizing handoff folders without writing code.
Pro tip: Write your preferences like you're briefing a smart new team member on your first day. The more context you give, the less you'll repeat yourself across conversations.
02 — Connectors

Connect your tools to Claude

Connectors (MCP) let Claude read and take actions inside your actual tools — not just talk about them. Here's what tentwenty should have connected.

🎨 Figma
Claude reads live design files, layer structure, tokens, and dev context. Core for MCP-ready handoff.
💬 Slack
Post design review summaries, feedback digests, and standup updates automatically from Claude.
📋 Monday.com
Create and update tasks from client feedback, track review cycles, and see team workload in real time.
📧 Gmail
Claude drafts client feedback replies, summarizes email threads, and creates tasks from client notes.
📅 Google Calendar
Schedule design reviews, user testing sessions, and sync sprints without switching apps.
🔧 Claude Code
For devs — terminal-level agent that reads Figma, writes code, runs skills and handles full implementation.
How to connect: Go to Claude.ai → Settings → Connectors. Each connector has a one-click install. For Figma MCP specifically, also install the Figma plugin from the Figma Community to enable the bridge.
03 — Memory & Preferences

Make Claude remember you

Claude can build a memory of your role, projects, preferences, and working style — so you don't start from scratch every conversation.

01
Turn on Memory
Go to Settings → Features → enable "Generate memory from chat history". Claude will start building context from your conversations automatically.
02
Tell Claude to remember
Just say: "Remember that I always work in Figma with Auto Layout, and I hand off to React developers." Claude stores this and uses it in future sessions.
03
Update or forget
Say: "Forget my old project context" or "Update — I've moved to a new project" and Claude will adjust. You can also manage memories in Settings → Memory.
Suggested memory to set for designers: Your tool stack, your design system (colors, fonts, spacing), your client's brand personality, and your most common output formats (Figma frames, dev-ready specs, etc).
04 — Sitemaps & Flows

Generate sitemaps & user flows

Use Claude to go from a product brief to a full information architecture in minutes — then bring it into Figma.

Generate
Generate a sitemap for a [type of product] targeting [user type]. Include primary nav, secondary pages, and any utility pages (login, error, empty states). Output as a visual diagram I can paste into Figma.
Use this first — gets you 80% of the IA in one shot.
Flows
Map the user flow for [key task e.g. "onboarding a new client"]. Show decision points, happy path, and failure states. Format it as a step-by-step flow I can diagram.
Add "include mobile and desktop variants" for responsive flows.
Refine
Here's the current sitemap: [paste]. Identify any missing pages, redundant flows, or UX dead-ends. Suggest a cleaner structure and explain the reasoning.
Great for cleaning up a client's existing confusing site structure.
05 — UX Research

UX research with Claude

From interview scripts to synthesis reports — Claude speeds up every stage of the research process.

Script
Write a 30-minute user interview script for [product context]. Focus on uncovering pain points around [topic]. Include warm-up questions, core tasks, and a debrief section.
Add "for [user type e.g. first-time users, enterprise admins]" to personalise the script.
Synthesize
Here are notes from [n] user interviews: [paste notes]. Synthesize the key themes, recurring pain points, and surprising insights. Format as a research report with a TL;DR summary at the top.
Use Qoqo.ai plugin in Figma alongside this for a visual output.
Document
Turn these research insights into a design brief: [paste insights]. Include user goals, frustrations, and 3 design principles this product should follow. Keep it to one page.
Hand this brief directly to designers before a sprint kickoff.
Skill to use: Connect the Kraftful plugin in Figma to bring user feedback directly into your design file. Pair it with Claude for synthesis.
06 — Moodboards & Design System

Moodboards, brand direction & design systems

Use Claude to decide design direction before touching Figma. Lock in the aesthetic, then let the system follow.

Aesthetic
Match the aesthetic of this image — the layout, the vibe, the visual hierarchy — but for my brand. Don't copy the content. Copy the style.
Your go-to for matching a reference without plagiarising it.
Lock
Keep the layout, color palette and composition exactly as they are. Change only the typography.
Useful when a client wants a refresh but not a redesign.
Design System
Based on this brand personality: [paste brief or adjectives], suggest a complete design system foundation — primary and secondary colors (with hex values), type scale, font pairings, spacing unit, and border radius style. Explain the reasoning behind each choice.
Then use Supa Palette plugin in Figma to build the color system visually.
Tokens
Here is our current design system: [paste tokens or description]. Identify any inconsistencies, missing tokens, or scalability issues. Suggest a clean token structure following design token best practices.
Use with Variable Inspector + Tokens Studio plugin for full audit.
Canvas Design Skill: Use the canvas-design skill in Claude to generate visual mockups and graphic directions before going into Figma. Great for client mood pitches.
07 — MCP-Ready UI

Designing UI that's ready for MCP

MCP-ready design means a developer can point Claude at your Figma file and get perfect code — no calls, no guessing. Here's how to design for it.

01
Use Auto Layout everywhere
Auto Layout translates directly to CSS flexbox. Any frame that uses it will generate near-perfect code. If you can't use Auto Layout, annotate why.
02
Annotate absolute positioning intent
Add a Dev Mode annotation for any frame using absolute positioning: "badge uses position:relative parent + absolute child". This one habit eliminates 90% of back-and-forth with devs after MCP output.
03
Name layers properly
MCP reads layer names as component names. Use Rename It plugin before handoff. Follow convention: ComponentName/variant (e.g. Button/Primary, Card/Hover).
04
Bind all styles to variables
No raw hex values or detached font styles. Every color, radius, shadow, and spacing value must be a variable. Run Variable Inspector to check before handoff.
05
Run pre-export checks
Run A11y contrast checker, Clean Document (remove unused styles), and check Tokens Studio shows no detached tokens. Only then share the Figma URL with devs.
Audit
Review this Figma design [URL] for MCP readiness. Flag any layers that aren't using Auto Layout, missing variable bindings, or naming issues that would cause problems during code generation.
Use with Figma MCP connected — Claude will actually read the file.
08 — Client Feedback

Processing client feedback with Claude

Turn messy, scattered client notes into clear action items — then resolve, document, and update in one workflow.

Triage
Here is raw feedback from our client: [paste]. Categorize it into: visual changes, UX/flow issues, content changes, and out-of-scope requests. For each item write one clear action point and an estimated effort (small/medium/large).
Paste this straight from a Slack thread, email, or Loom transcript.
Resolve
The client said: "[exact quote]". In context of a [product type] design, what is the underlying concern here? Suggest 2 ways to address it — one that stays within scope, one that goes deeper.
Useful when client feedback is vague or contradictory.
Reply
Write a professional reply to this client feedback: [paste]. Acknowledge their points, explain what we'll action, what we recommend against and why, and set a timeline. Tone: confident and collaborative, not defensive.
Use with Gmail connector to send directly from Claude.
Automation: Connect Monday.com + Slack — Claude can triage client feedback, create Monday tasks with priority tags, and post a summary to your team Slack channel automatically.
09 — User Testing

User testing support with Claude

Claude helps you plan tests, analyze session recordings, and synthesize findings into design decisions.

Plan
Create a usability test plan for [feature/flow]. Include: test objectives, tasks for participants, success metrics, and what we'll do if a task fails. Keep it concise — one page max.
Pair with ProtoPie or Figma prototype for the test itself.
Analyze
Here are observations from [n] usability sessions: [paste notes or transcript]. Identify the top 3 usability problems, rank them by severity, and suggest a specific design fix for each.
Works great with Loom transcripts — copy/paste the auto-transcript.
Report
Turn these usability findings into a short report for stakeholders: [paste]. Include: what we tested, what we found, what we're going to change, and what we're monitoring. Non-technical audience.
Share directly with clients or PMs — no design jargon.
10 — Dev Handoff

Flawless handoff to developers

The goal: devs open the Figma URL, run Claude Code with MCP, and get production-ready code with zero back-and-forth.

Designer prepares ✓
  • Auto Layout on all frames
  • Absolute position annotations added
  • All styles bound to variables
  • Layer names follow component convention
  • A11y contrast check passed
  • Clean Document run — no unused styles
  • Dev Mode notes filled on ambiguous frames
Developer receives ✓
  • Clean Figma URL shared
  • UI/UX Pro Max skill installed in Claude Code
  • Figma MCP connector active
  • Implement Design skill loaded
  • Component naming matches codebase conventions
  • SKILL.md shared with design system context
  • Any edge cases documented in Dev Mode
Implement
Use the implement-design skill and Figma MCP. Here's the Figma URL: [url]. Give me a production-ready React component. Match all spacing, colors, font styles, and states exactly as designed. Note any edge cases.
Run in Claude Code with both the UI/UX Pro Max skill and Figma MCP active.
Fix Gap
The implementation differs from the Figma design in these ways: [describe diff]. The design intent was [explain]. Please update the code to match. Here is the relevant Figma frame: [url].
For iteration after the first MCP pass — closes the pixel gap.
Skill install for devs: In Claude Code terminal, say: "Download this skill for me: https://github.com/nextlevelbuilder/ui-ux-pro-max-skill" — this teaches Claude your color palette, fonts, spacing and UX guidelines before writing a line of code.

12 — Project Workflow

Claude × Figma across the tentwenty project workflow

Every phase of a website project — from brief to launch — mapped to exactly where Claude and Figma work together. Use this as your day-to-day reference.

🔍
Phase 1 · Discovery & Brief
Kickoff, brief analysis & stakeholder goals
Client brief comes in — goals, audience, competitors, scope. Translate a raw document into a clear design direction fast.
Claude does
Paste the client brief → Claude extracts design goals, audience personas, and constraints. Ask it to identify gaps before your kickoff call. Summarise competitors into a structured table.
Summarise this brief into goals + constraints What questions should I ask the client? Analyse these 3 competitor sites
Claude Projects Deep Research
🧪
Phase 2 · UX Research
User research, personas & journey maps
Building empathy for the end user before touching Figma. Interview scripts, personas, pain-point mapping.
Claude + Figma (QoQo plugin) does
Claude drafts interview scripts and survey questions. Use QoQo.ai Figma plugin to generate personas, journey maps and affinity maps directly onto a FigJam board. Claude synthesises interview notes into key insights.

Phase 3 · Sitemap & IA
Information architecture & page structure
Deciding what pages exist, how they connect, what content lives where. Often goes back-and-forth with the client.
Claude does
From the brief + research, Claude generates a full sitemap with page names, hierarchy and content notes. Drop it into FigJam as a structured list. Claude also drafts content outlines per page.

Visual language, references & tone
Establishing the visual feel before committing to a design system. Colour direction, typographic mood, reference gathering.
Claude + Figma does
Claude generates detailed visual direction prompts — brand adjectives, colour palette rationale, typographic pairings. Use Figma AI Canvas Agent to turn those into moodboard frames directly on canvas.
Describe a visual direction for [brand adjectives] Suggest 3 type pairings with rationale Generate a colour palette for [mood]

Tokens, components & style foundations
Building the single source of truth — colours, type scale, spacing, component library — before any screens are designed.
Claude + Figma MCP does
Claude generates token naming conventions, spacing scales and component specs. With Figma MCP connected to Claude Code, it reads your variables directly and writes a SKILL.md that teaches devs the full system.

Lo-fi flows → hi-fi screens
Translating IA and content outlines into actual layouts. Moving from wireframes to polished, component-driven screens.
Claude + Figma does
Use UX Pilot Figma plugin to prompt-generate wireframe frames. Claude writes UX copy, microcopy, and placeholder content matched to the brand voice. Ask Claude to critique layouts for hierarchy and accessibility before moving to hi-fi.

Presenting, handling feedback & iterating
Sharing screens for approval, collecting structured feedback, making sense of conflicting notes, and turning it into clear revision tasks.
Claude does
Claude writes the presentation narrative for each screen — explaining design decisions in client language. Paste raw feedback → Claude organises it into prioritised revision tasks and flags conflicting requests.
